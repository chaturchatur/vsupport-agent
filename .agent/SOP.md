# Standard Operating Procedures

> Last verified: 2026-03-17

## Documentation Workflow

1. **Initial setup:** Run `/init-docs` to deep-scan the codebase and generate the `.agent/` documentation structure from scratch.
2. **Ongoing updates:** After making code changes, run `/update-docs` to selectively update only the relevant documentation files.
3. **Always read first:** Before modifying any `.agent/` doc, read `.agent/README.md` to understand what exists and avoid duplication.
4. **Consolidate:** Keep documentation consolidated — avoid overlap between files. If content fits in `System.md`, don't create a separate file for it.
5. **Cross-references:** When creating new doc files, include a "Related docs" section listing relevant files for full context.

## Testing

- **Run tests:** `python3 -m pytest tests/ -v -p no:playwright`
- The `-p no:playwright` flag is required because a globally installed `pytest-playwright` plugin conflicts with this project's test runner.

## Coding Standards

- **Phone numbers**: Always store and compare in E.164 format (e.g., `+14155551234`). Phone normalization for lookups is handled in n8n Code nodes (JS-based).
- **Config**: All env vars loaded via `app/config.py` (Pydantic Settings). Never hardcode secrets.

## VAPI Deployment

- **Deploy script**: `./scripts/deploy-vapi.sh` — resolves `{{placeholders}}` in `assistant_config.json` and deploys via REST API.
- **VAPI CLI limitations**: The `vapi assistant create` command is interactive-only (no `--body` flag). Use the REST API (`POST https://api.vapi.ai/assistant`) instead.
- **API key**: Use the private key from VAPI dashboard (same as what `vapi auth` stores in `~/.vapi-cli.yaml`). The `.env` `VAPI_API_KEY` must match.
- **Config gotchas** (learned during Phase 7):
  - Tools go in `model.tools[]`, NOT at the top level of the assistant config
  - Webhook auth uses `server.headers: {"x-vapi-secret": "..."}`, NOT `server.secret`
  - Summary config is `analysisPlan.summaryPrompt` (flat string), NOT `analysisPlan.summaryPlan.prompt`
- **Resolved config**: `vapi/assistant_config.resolved.json` is gitignored (contains secrets). Regenerated by deploy script.

## n8n Workflow Management

- All workflows created/managed via n8n REST API (`N8N_API_KEY`).
- Workflow JSON exports stored in `n8n/workflows/` for version control.
- Webhook auth uses Header Auth credential (`x-vapi-secret` header).
